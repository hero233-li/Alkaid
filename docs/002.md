可以。你现在要建立一个固定思路：

> **新增一个前端页面，不等于把新接口逻辑继续塞进 `product_data/tasks.py`。**
> 正确做法是：新页面拥有自己的“业务链路”，但复用 Alkaid 已有的 Job、日志、进度、重试、取消和 HTTP 基础设施。

前端的 Axios 基础路径是 `/api`。
提交接口已经写成 `/product-data/tools/application-links/generate`。
最终页面会读取：

```ts

```

但是当前后端的：

```text
Alkaid-python/apps/product_data/urls.py
```

只注册了：

```text
applications/config
applications
```

还没有申请链接接口，所以现在这条前后端链路实际上没有接完整。

# 五、第三步：View 只负责创建 Job

新增：

```text
apps/product_data/application_links/views.py
```

大致这样：

```python
import uuid

from django.conf import settings
from django.db import transaction
from django.http import (
    HttpRequest,
    JsonResponse,
)
from django.views.decorators.csrf import (
    csrf_exempt,
)
from django.views.decorators.http import (
    require_POST,
)
from pydantic import ValidationError

from apps.core.responses import (
    api_error,
    api_response,
)
from apps.jobs.services import (
    JobConflict,
    JobRepository,
    serialize_job,
)

from .schemas import (
    ApplicationLinkSubmission,
)
from .tasks import (
    execute_application_link,
)


@csrf_exempt
@require_POST
def generate_application_link(
    request: HttpRequest,
) -> JsonResponse:

    try:
        submission = (
            ApplicationLinkSubmission
            .model_validate_json(
                request.body
            )
        )

    except ValidationError as exc:
        return api_error(
            f"申请链接参数无效：{exc}",
            status=400,
        )

    idempotency_key = (
        request.headers.get(
            "X-Idempotency-Key"
        )
        or str(uuid.uuid4())
    )

    trace_id = (
        request.headers.get(
            "X-Trace-ID"
        )
        or uuid.uuid4().hex
    )

    try:
        created = JobRepository.create(
            kind=(
                "application_link_generation"
            ),

            name=(
                f"申请链接生成-"
                f"{submission.product}"
            ),

            product=submission.product,

            payload=(
                submission.model_dump(
                    mode="json"
                )
            ),

            trace_id=trace_id,

            idempotency_key=(
                idempotency_key
            ),

            timeout_seconds=(
                settings
                .APPLICATION_LINK_TIMEOUT_SECONDS
            ),
        )

    except JobConflict as exc:
        return api_error(
            str(exc),
            status=409,
        )

    if created.created:
        transaction.on_commit(
            lambda:
            execute_application_link.delay(
                created.job.id
            )
        )

    return api_response(
        serialize_job(
            created.job
        ),
        status=(
            202
            if created.created
            else 200
        ),
    )
```

这个 View 不应该直接：

```python
requests.post(...)
```

也不要直接：

```python
生成链接
```

它只做四件事：

```text
1. 接收参数

2. 校验参数

3. 创建 Job

4. 把 Job 投递给 Task
```

跟现在产品申请的 View 是同一模式。现有产品申请也是创建 Job 后，通过 `transaction.on_commit()` 投递 Celery。

# 七、第四步：新增申请链接 Task

新增：

```text
apps/product_data/application_links/tasks.py
```

写法和产品申请类似：

```python
from celery import shared_task
from celery.exceptions import (
    SoftTimeLimitExceeded,
)
from django.conf import settings
from django.utils import timezone

from apps.jobs.models import (
    JobStatus,
)
from apps.jobs.services import (
    InvalidJobTransition,
    JobRepository,
)

from .schemas import (
    ApplicationLinkSubmission,
)
from .services import (
    ApplicationLinkGenerator,
)


@shared_task(
    bind=True,
    name=(
        "apps.product_data."
        "application_links.tasks."
        "execute_application_link"
    ),
    acks_late=True,
    reject_on_worker_lost=True,
    soft_time_limit=(
        settings
        .APPLICATION_LINK_TIMEOUT_SECONDS
    ),
    time_limit=(
        settings
        .APPLICATION_LINK_TIMEOUT_SECONDS
        + 10
    ),
)
def execute_application_link(
    self,
    job_id: int,
) -> None:

    task_id = str(
        self.request.id
        or "local-eager-task"
    )

    job = JobRepository.mark_running(
        job_id,
        task_id,
    )

    if job is None:
        return

    try:
        if (
            job.deadline_at
            and
            job.deadline_at
            <= timezone.now()
        ):
            JobRepository.mark_timed_out(
                job.id,
                "申请链接任务等待超时",
            )
            return

        submission = (
            ApplicationLinkSubmission
            .model_validate(
                job.payload
            )
        )

        JobRepository.update_progress(
            job.id,
            stage="validate",
            progress=30,
            message=(
                "申请链接参数校验完成"
            ),
        )

        job.refresh_from_db()

        if (
            job.status
            ==
            JobStatus.CANCEL_REQUESTED
        ):
            JobRepository.mark_cancelled(
                job.id
            )
            return

        result = (
            ApplicationLinkGenerator()
            .generate(
                job,
                submission,
            )
        )

        JobRepository.update_progress(
            job.id,
            stage="generate",
            progress=90,
            message=(
                "申请链接生成完成，"
                "正在保存结果"
            ),
        )

        JobRepository.mark_success(
            job.id,
            {
                "links": result
            },
        )

    except InvalidJobTransition:
        JobRepository.mark_cancelled(
            job.id
        )

    except SoftTimeLimitExceeded:
        JobRepository.mark_timed_out(
            job.id
        )
        raise

    except Exception as exc:
        JobRepository.mark_failed(
            job.id,
            (
                f"{type(exc).__name__}: "
                f"{exc}"
            ),
        )
        raise
```

这里最重要的是：

```python
JobRepository.mark_success(
    job.id,
    {
        "links": result
    },
)
```

不能只写：

```python
JobRepository.mark_success(
    job.id,
    result,
)
```

因为当前前端取的是：

```ts
job.result.links
```

后端最终 Job 应该长这样：

```json
{
  "status": "success",
  "progress": 100,
  "result": {
    "links": {
      "internalUrl": "http://xxx/internal",
      "externalUrl": "https://xxx/external",
      "generatedAt": "2026-07-10T23:30:00"
    }
  }
}
```

前端才能正常显示。

# 十一、真正的接口先后顺序放在哪里

假设申请链接需要：

```text
1. 登录接口

2. 查询客户接口

3. 生成申请编号

4. 生成内部申请链接

5. 生成外部申请链接
```

那么应该写在 Handler：

```python
class ProductADynamicLinkHandler:

    code = (
        "product_a_dynamic_link_v1"
    )

    def generate(
        self,
        job,
        submission,
    ):

        token = self.login(
            job,
            submission,
        )

        customer = (
            self.query_customer(
                job,
                submission,
                token,
            )
        )

        application_no = (
            self.create_application(
                job,
                submission,
                customer,
                token,
            )
        )

        internal_url = (
            self.create_internal_link(
                job,
                application_no,
                token,
            )
        )

        external_url = (
            self.create_external_link(
                job,
                application_no,
                token,
            )
        )

        return {
            "internalUrl": (
                internal_url
            ),

            "externalUrl": (
                external_url
            ),

            "generatedAt": (
                timezone.now()
                .isoformat()
            ),
        }
```

这里就是：

```text
登录

↓

查询客户

↓

创建申请

↓

生成内部链接

↓

生成外部链接
```

顺序执行。

前一个失败：

```text
后面的不执行

↓

异常抛给 Task

↓

Task 调用 mark_failed()

↓

前端轮询得到 failed
```

```python
    )
```

所以申请链接任务第一次可以正常执行。

但是用户点击：

```text
重试
```

会报：

```text
不支持的任务类型：
application_link_generation
```

必须增加：

```python
def _enqueue(
    job: Job,
) -> None:

    if (
        job.kind
        == "product_application"
    ):
        from apps.product_data.tasks import (
            execute_product_application,
        )

        execute_product_application.delay(
            job.id
        )

        return

    if (
        job.kind
        ==
        "application_link_generation"
    ):
        from (
            apps.product_data
            .application_links.tasks
        ) import (
            execute_application_link,
        )

        execute_application_link.delay(
            job.id
        )

        return

    raise ValueError(
        f"不支持的任务类型："
        f"{job.kind}"
    )
```

后续任务多了以后，可以把这个继续抽成 Job Task 注册表。

# 十四、增加独立超时时间

现在设置里只有：

```python
PRODUCT_APPLICATION_TIMEOUT_SECONDS
```

建议新增：

```python
APPLICATION_LINK_TIMEOUT_SECONDS = int(
    os.getenv(
        "APPLICATION_LINK_TIMEOUT_SECONDS",
        "120",
    )
)
```

然后 `.env.example` 增加：

```env
APPLICATION_LINK_TIMEOUT_SECONDS=120
```

这样：

```text
产品申请超时：300 秒

申请链接生成：120 秒
```

不会绑死在一起。

# 十五、申请链接要不要保存“执行配置快照”

我认为需要。

因为你现在申请链接已经出现：

```text
产品 A：

动态链接需要字段：
客户名称
申请区域
金额

产品 B：

动态链接需要字段：
客户手机号
企业名称

产品 C：

只支持太阳码
```

未来配置一定会继续变化。

建议创建 Job 时同时保存：

```json
{
  "configVersion": 1,
  "product": "产品A",
  "category": "动态链接",
  "handler": "product_a_dynamic_link_v1",
  "requiredFields": [
    "customerName",
    "applicationArea"
  ]
}
```

保存到：

```python
execution_config_snapshot
```

现有 Job 已经有这个字段。

那么：

```text
今天创建的任务

↓

使用今天的配置

↓

明天修改配置

↓

今天失败的任务点击重试

↓

仍使用创建时的旧配置
```

不会出现：

```text
第一次执行一种流程

重试突然变成另一种流程
```

这个机制现有产品申请已经在用，申请链接也应该沿用。

```text
按钮

loading

显示结果
```
